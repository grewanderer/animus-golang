# Консоль управления Animus Datalab

## Архитектура и границы доверия

Консоль реализована отдельным приложением в `closed/frontend_console` и обращается **только** к Gateway API. Прямые вызовы внутренних сервисов (Experiments, Dataset Registry, Lineage, Audit, SIEM) запрещены. Все обращения проходят через:

- `/auth/*` для управления сессиями;
- `/api/*` прокси‑маршруты (Dataset Registry, Experiments, Lineage, Audit);
- прямые gateway‑эндпоинты для моделей и health‑probes.

Контрольная плоскость не исполняет пользовательский код. Все запросы фиксируются в аудите на серверной стороне.

## Аутентификация и RBAC

- Консоль проверяет текущую сессию через `/auth/session` и отображает состояние в верхней панели.
- При отсутствии сессии выводится детерминированное сообщение и ссылка на `/auth/login`.
- UI отражает доступы (RBAC) через отключённые действия и пояснения, но **никогда** не полагается на фронтенд‑ограничения — сервер остаётся источником истины.
- Любые ошибки авторизации отображаются с кодом и `request_id`.

## UX‑принципы

- Все сущности, фиксируемые сервером, помечаются как **НЕИЗМЕНЯЕМО** и сопровождаются хэшами.
- Ошибки не приводят к «тупикам»: сообщается код, `request_id`, и даётся краткая рекомендация.
- Долгие операции регистрируются в панели фоновых операций; состояния фиксированы, без неинформативных спиннеров.
- Быстрые действия доступны из верхней панели.

## Запуск, сборка, тестирование

Выполняется из `closed/frontend_console`.

- Dev‑режим:
  - `npm install`
  - `npm run dev`
- Сборка:
  - `npm run build`
- Типы и тесты:
  - `npm run typecheck`
  - `npm run test`

Консоль не инициирует сетевые серверы в тестах; все проверки являются оффлайн‑детерминированными.

## Стиль и дизайн‑токены

Визуальный язык полностью наследует `closed/frontend_landing`:

- базовые токены цветов, типографики, радиусов и сеток перенесены в `closed/frontend_console/app/globals.css`;
- компоненты используют те же принципы плотной табличной верстки;
- декоративные элементы и иллюстрации отсутствуют — интерфейс ориентирован на операционный контур.

## Примечания по эксплуатации

- Все URL‑ссылки строятся относительно Gateway (`NEXT_PUBLIC_GATEWAY_URL`), при отсутствии переменной используется относительный путь.
- DevEnv‑сессии доступны только через прокси‑маршрут `/devenv-sessions/{session_id}/proxy/*`.
