# ADR-0012: Точка принудительной проверки подписи образов

**Статус:** Принято

**Дата:** 2026-02-04

## Контекст
Образы в `EnvLock` уже закрепляются по digest. Требуется добавить опциональную проверку подписи (cosign‑like) и определить точку принудительного контроля без изменения семантики исполнения.

## Решение
- Принудительная проверка подписи выполняется при создании EnvironmentLock (CP‑слой), до фиксации блокировки окружения.
- Проверка digest сохраняется как обязательная, проверка подписи включается политикой проекта (deny_unsigned/allow_unsigned).
- Результат проверки сохраняется и аудируется вместе со снимком политики.

## Обоснование
- Проверка на этапе создания EnvLock обеспечивает воспроизводимость и предотвращает закрепление неподтверждённых образов.
- Исключается необходимость выполнения проверки в DP, сохраняя принцип «CP не выполняет пользовательский код».
- Политика остаётся детерминируемой и проверяемой офлайн через сохранённый результат.

## Последствия
- Ошибки верификации подписи блокируют создание EnvLock (deterministic fail‑closed).
- Для восстановления требуется обновление политики или корректная подпись образа.
